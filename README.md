# АРЧА — сайт

Одностраничный сайт-визитка для кафе **АРЧА** (Токмок, Кыргызстан). «Кафе среди хвойного воздуха» — европейская, восточная, национальная и турецкая кухни.

Стек: **Vite + React 18 + Tailwind CSS**. Без бэкенда — статика, разворачивается на GitHub Pages (или любом другом статик-хостинге).

## Запуск

```bash
pnpm install      # один раз, поставит зависимости
pnpm dev          # http://localhost:5173 — режим разработки
pnpm build        # продакшен-сборка в dist/
pnpm preview      # предпросмотр продакшен-сборки
```

(Можно использовать `npm` или `yarn` — сработает аналогично.)

## Структура

```
src/
  App.jsx                 — корневой компонент, собирает все секции
  main.jsx                — точка входа React
  index.css               — Tailwind + базовые утилиты (.container-x, .display, .pill, .tap)
  data/
    menu.js               — категории и позиции меню (демо для заказчика)
  components/
    Header.jsx            — фиксированная шапка с навигацией и моб. меню
    Hero.jsx              — первый экран: АРЧА / эмодзи / КАФЕ
    Menu.jsx              — меню с категориями-пилюлями
    Promo.jsx             — тёмно-зелёный callout-блок с акцией
    Footer.jsx            — контакты, часы, карта OSM
    Logo.jsx              — SVG-бейдж АРЧИ
    DishImage.jsx         — картинка блюда с плейсхолдером
    Reveal.jsx            — fade-up анимация при скролле
  lib/
    useInView.js          — хук для Reveal

public/
  favicon.svg
  images/
    hero.png              — НЕ используется в текущем Hero (плейсхолдер), оставлено для будущей замены
    promo.png             — НЕ используется в текущем Promo (плейсхолдер)
    menu/                 — фото блюд: <imgKey>.png|jpg
```

## Палитра и шрифты

В `tailwind.config.js`:

- `forest-800` `#1F2A1E` — основной тёмный (Hero, Promo, активные пилюли)
- `sand-50` / `sand-100` — кремовые тона (фоны светлых секций, страница в целом)
- `gold-400` `#D4A84B` / `gold-500` `#BB8E2E` — золотые акценты (обводка лого, кнопки)
- `ink-800` / `ink-900` — тёмный текст на светлых фонах

Шрифты подключены через Google Fonts в `index.html`:

- **Fraunces** — display (заголовки, бренд)
- **Inter** — body
- **Caveat** — script-акценты

## Как править контент

- **Меню (блюда, цены, описания, теги)** — `src/data/menu.js`
- **Тексты Hero / Promo** — `src/components/Hero.jsx`, `src/components/Promo.jsx`
- **Часы работы / телефон / адрес** — `src/components/Footer.jsx` (и `src/components/Header.jsx` для мобильного меню)
- **Цвета бренда / шрифты** — `tailwind.config.js`

## Как добавить свои фотографии

Все картинки опциональны: пока файла нет — показывается стилизованный плейсхолдер с эмодзи. Положить файл в правильное место — он подхватится автоматически.

### Hero и Promo

Сейчас оба используют эмодзи-плейсхолдеры (☕ и 🍰), фото намеренно не подключены. Если захочется добавить — нужно править `Hero.jsx` / `Promo.jsx`, вернув `<img>`-тег по примеру La Cafe.

### Блюда меню

В `src/data/menu.js` у каждого блюда есть поле `imgKey`. Файл должен лежать как `public/images/menu/<imgKey>.png` (или `.jpg`).

Сейчас лежат демо-фото от предыдущего проекта (La Cafe) — это нормально, заменим на реальные блюда АРЧИ позже.

## Деплой

В репозитории есть GitHub Actions workflow (`.github/workflows/deploy.yml`), который при пуше в `main` автоматически собирает сайт и публикует на GitHub Pages.

Что нужно сделать один раз:

1. Создать репо `archa` (или другое имя — тогда обновить `base` в `vite.config.js`).
2. В настройках репо: **Settings → Pages → Build and deployment → Source** → выбрать `GitHub Actions`.
3. Запушить ветку `main` — workflow развернёт сайт на `https://<username>.github.io/archa/`.

Готовая сборка также лежит в `dist/` (после `pnpm build`) — можно залить на любой статик-хостинг (Vercel, Netlify, Cloudflare Pages, nginx).
